# 知识库计划

第一阶段保留标准库轻量索引，保证无依赖可运行。

第二阶段已开始接入可替换知识库协议：

- 当前实现：`knowledge-build` / `knowledge-search`。
- 当前后端：标准库 JSON，输出 `memory/knowledge_store.json`。
- 元数据：来源、类型、章节、角色、确认状态、权威级别。

第三阶段接入向量知识库：

- 文档摄取：作品正文、canon、人物、风格样本、审查报告。
- 元数据：路径、类型、角色、章节、时间线、确认状态。
- 检索：关键词 + 向量混合召回。
- 约束：召回结果必须带来源，不自动写回 canon。

第四阶段接入图谱：

- `plot/longform_graph.json` 作为 Neo4j 导入前的稳定中间格式。
- 图谱用于因果、关系、伏笔、场景顺序和冲突检查。
