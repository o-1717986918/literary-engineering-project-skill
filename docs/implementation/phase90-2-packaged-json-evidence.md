# Phase 90.2：JSON 证据包装与连接设置降级

本阶段修正前端表达：JSON 信息不应该被隐藏，只是不应该以调试格式裸露给用户。

## 改动

1. 项目总控新增“项目证据柜”。
2. 证据柜读取 `workflow-dashboard` 返回的 JSON 字段，并包装为中文卡片：
   - 状态机读数
   - 平台任务读数
   - 路线审计读数
   - 输出与索引路径
   - 使用边界
   - 最近活动
3. 原始 JSON 不在页面中直接展开；机器索引路径仍作为可追溯证据展示。
4. “模型配置”不再作为一级导航模块出现。
5. API Key、API 地址、模型名、温度和默认目录配置保留在总控页“连接设置”折叠面板中。

## 架构判断

当前项目的正式创作架构是“平台 Agent + CLI 状态机”。平台 Agent 提供推理、生成、审查和自由决策能力，CLI 负责发放任务、收束产物、写入状态、验证门禁。

因此，单独的模型配置不应被设计成主交互模块。它仍然有价值，但定位应是高级连接能力：

- 本地 `serve-api` 调试真实 provider。
- 接入 DeepSeek 等 HTTP Chat API。
- 为 Dify / LangGraph 等外部编排预留统一配置。
- 给非 Codex / Claude 宿主环境提供备用 LLM 通道。

## 验证

- 前端仍保留显式 API Key 输入能力。
- 页面不包含 `data-view="config"` 和 raw dashboard JSON 面板。
- `/workflow/dashboard` 返回的摘要、路径、规则和事件均进入用户可读展示。
