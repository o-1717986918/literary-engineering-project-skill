# Phase 96 - Swiss Editorial Frontend

本阶段把前端视觉从“能用的项目后台”升级为更有记忆点的文学工程总控台。设计目标不是制造装饰感，而是让用户更快看懂：作品正文在哪里、流程卡在哪里、证据是否可靠、下一步该做什么。

## 设计锚点

采用已安装的 `frontend-design-anchors` 中的 Swiss 方向：

- 纯白 / 中性浅灰页面底色。
- Helvetica Neue / Microsoft YaHei UI / Arial 单一无衬线字体系统。
- 黑色 1px hairline、可见网格、大号页码数字。
- Yves Klein Blue 作为主行动色，红色只用于正文主模块、阻塞或审查提醒。
- 不使用暖纸、柔和阴影、圆角卡片、装饰性渐变或模拟代码感文案。

本项目是严肃的长篇创作工程工具，视觉上的“惊艳”来自信息秩序和版式力量，而不是营销式插画或炫光背景。

## 已实现

1. 总体视觉系统
   - `frontend/styles.css` 改为黑白网格、硬边框、硬阴影和大号数字构图。
   - 顶部 hero、侧边栏、面板、指标块、证据卡、任务卡统一为编辑部版式。
   - 清理主要界面的圆角、柔阴影和暖色背景，减少模板化后台观感。

2. 项目总控更醒目
   - “已完成正文”继续作为项目总控的显眼主模块。
   - 正文预览使用稿纸式滑动阅览框，正文与工程证据视觉上明确分离。
   - 指标块使用大号 tabular numerics，更像项目状态报表。

3. 作品档案更亲用户
   - 档案封面、分类统计、详情头、事实卡、备注卡都统一为资料夹式布局。
   - JSON / YAML 信息继续由前端包装成卡片、标签、摘要和阅读框，不直接暴露原始结构。
   - 分支、文风、审查、字数预算等内容仍按“作品资料”展示，而不是程序文件清单。

4. 人类选择节点更清楚
   - 分支卡、推荐理由、最近选择和安全备注统一为硬边卡片。
   - 用户仍只在前端提交结构化选择或备注，正式推进继续由平台 Agent 通过 CLI 状态机完成。

5. 版本一致性
   - 前端 `UI_VERSION`、包版本和 `/health` 版本同步为 `0.97.0`。

## 边界

- 前端仍不直接写正式正文、canon、角色状态、发布指针或审查结论。
- 视觉升级不改变 CLI gate、route audit、review、promotion、export readiness 等正式链路。
- 本页不是替代平台 Agent 的创作总监，而是让用户能看懂平台 Agent 和 CLI 状态机正在做什么。

## 验证要求

- `python -m unittest discover -s tests -q`
- `python -m literary_engineering_workbench prompt-registry-validate`
- `git diff --check`
- 浏览器检查桌面和移动端：
  - 无横向溢出；
  - 主导航、项目总控、已完成正文、作品档案、选择节点均可读；
  - 正文滑动阅览框正常；
  - `/health` 版本与前端版本一致。
