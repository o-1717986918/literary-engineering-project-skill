# Phase 74：生成层反 AI 腔硬约束与语义清洗边界

本阶段处理一个真实 review 漏洞：正文生成后再用脚本清理“不是……而是……”或破折号转折，容易把有效修辞改坏，甚至造成语义反转。正确位置不是批量清洗层，而是生成前约束、文风 prompt、平台 AgentReview 和人工/Agent 语义修订闭环。

## 设计判断

- “不是 A，而是 B”并不天然是错误。它可能是人物认知二分、叙述者纠偏、信息反转、讽刺顿挫或某个授权文风的核心修辞。
- 模型默认喜欢把这类结构当作文学化模板，所以生成层必须默认禁用，不能只写“不要高频”。
- 破折号变体如“不是 A——是 B”“不是 A。是 B”与“不是 A，而是 B”属于同一类机械对照风险，不能被标点变化绕过。
- 脚本没有语义理解能力。脚本可以提示风险、统一安全排版，但不能删除否定、批量删除破折号、替换心理判断或改写纠偏结构。

## 已实现

- `anti_ai_style.py`：AI trace prompt 改为“默认禁用、明确例外”；lint 能识别破折号/句号/逗号替代“而是”的变体。单次出现标为 low 语义复核，多次出现标为 medium 修订问题。
- `prompt_pack.py` 与 `templates/prompts/scene_generation_system.md`：生成前硬约束默认禁止机械对照句式和破折号变体；Style Skill 或用户明确授权时才允许功能性例外。
- `punctuation_standard.py` 与 `references/punctuation-standard.md`：区分功能性破折号和机械装饰，新增自动清洗边界。
- `style_prompt.py` 与 `style_prompt_agent.py`：文风学习输出必须说明高破折号、否定纠偏等特征的适用语境、禁用边界和审查方法，避免把模型习惯误写成作家风格。
- `scene_draft.py` 与初始化说明：新建项目和草稿工作台继承同一规则。
- 回归测试覆盖“不是 C 营的——是 E 营的人”这类可疑但可能有效的结构，确保系统提示语义复核，而不是脚本删改。

## 后续原则

新增任何文本清洗脚本时，必须遵守本阶段边界：只做安全排版规范化或风险报告，不做语义级自动改写。需要语义判断的修改进入 `revise-scene` 或平台 Agent 逐句编辑。
