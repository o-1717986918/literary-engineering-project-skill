# Phase 75：朴素叙述约束与 AI 腔密度门禁

本阶段把“降低 AI 腔”从单纯提示词要求升级为生成、文风学习、审查和文档一致的分层规则。

## 设计判断

- 机械“不是……而是……”“并非……而是……”“与其说……不如说……”和“不是……——是……”等变体是核心禁区。平台 Agent 不应把它们解释为合理修辞，也不应让 Style Skill 授权模型复用这些句式模板。
- 若源文风里存在否定纠偏或强停顿，应提取其叙事功能：认知二分、信息反转、讽刺顿挫、叙述者纠偏、话语中断。落到生成时，应改为动作、事实顺序、信息差或直接陈述。
- 大词表不适合零容忍。器官轮岗、万能占位、比喻依赖、抽象总结、景物强制同步、模板转折和金句化收束采用约 2% 叙事单元密度门禁：孤立命中是低级复核信号，密集命中升级为修订问题。
- 破折号在朴素叙述中是强复核信号。孤立出现需要语义判断；超过约 2% 密度、同段反复出现，或替代“而是/但是/于是”时必须修订。
- 可挂载文风 prompt 的上限提升到 2500 字，以容纳更完整的身份、边界、风格机制、句法、标点、AI 腔门禁和自检规则。

## 已实现

- `anti_ai_style.py`：新增大词表风险检测、2% 密度门禁、机械对照核心禁区、破折号/逗号链/模板句式/比喻依赖检测。
- `prompt_pack.py`、`templates/prompts/scene_generation_system.md`、`scene_draft.py`：把朴素叙述、核心禁区和密度门禁注入生成前标准。
- `style_prompt.py`、`style_prompt_agent.py`、`platform_agent_tasks.py`：文风学习输出上限提升到 2500 字，并要求 Style Skill 抽象功能而不是授权模板句式。
- `punctuation_standard.py` 与 `references/punctuation-standard.md`：将破折号从“可授权高密度风格”调整为强复核信号。
- `SKILL.md`、`AGENTS.md`、`references/*`、README 和前端文案：同步 500-2500、核心禁区和 2% 密度门禁。
- 回归测试覆盖机械对照不可合理化、风险词孤例低级复核、密集风险词中等修订、破折号和长逗号链检测。

## 后续原则

不要把禁词表当成自动改写表。脚本只能报告风险或做安全排版规范化；真正的句子修订应进入 `revise-scene` 或平台 Agent 逐句编辑，保证语义、人物意图和文风功能不被误删。
