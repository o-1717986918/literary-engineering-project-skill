# Phase 90.1：中文化、人性化总控前端

本阶段重做本地前端控制台，让它更像“文学项目驾驶舱”，而不是调试后台。

## 改动

1. 前端导航收束为三块：
   - 项目总控
   - 文风挂载
   - 模型配置
2. 移除旧的创作总监聊天页和大段文风学习操作表单。
3. 项目总控不再直接展示原始 JSON。
4. route gate、sidecar、next action、recent events 被翻译成中文卡片、摘要和下一步建议。
5. 配置页改为脱敏连接画像，不再用 raw JSON 展示配置。

## 设计原则

- 用户看到的是“项目是否能继续推进”，不是内部字段名。
- 技术证据仍可追溯，但默认以中文包装。
- 前端只读展示状态，不替代 CLI 状态机，也不直接写 completion marker、review、promotion 或 state patch。
- 文风挂载页只展示挂载状态；文风学习、语料导入和提示词生成仍建议由正式 CLI route 管理。

后续 `v0.90.2` 将“模型配置”从一级导航降级为总控页内的“连接设置”，并把 dashboard JSON 扩展为证据卡展示。`v0.90.1` 保留在这里作为初版前端收束记录。

## 验证

- 前端静态资源测试覆盖新导航、新 dashboard endpoint、无旧创作总监入口、无 raw JSON 面板。
- 浏览器检查无控制台错误。
