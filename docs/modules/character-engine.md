# 人物引擎

人物档案至少包含：

- 目标、信念、欲望、恐惧
- 已知事实和误解
- 关系网络
- 道德边界
- 背景故事：只作为隐性行为因果，不应在正文中直白说明
- 语言习惯
- 当前状态和成长弧

角色推演时，Agent 只能根据人物处境和认知行动，不能为了剧情方便强行 OOC。

## 背景故事使用规则

`background_story` 维护角色过去经历如何塑造现在行为。它影响：

- 角色会优先注意什么。
- 角色会回避什么。
- 角色何时误判、迟疑或过度反应。
- 角色说话的克制、遮掩或防御方式。
- 角色在关键剧情分支中愿意承担哪类代价。

它不应直接变成正文里的解释性设定段落。除非场景目标明确要求揭示过去，否则背景故事只通过选择、动作、语气、关系压力和潜台词体现。

## 状态演化

场景完成后，使用 `state-evolve` 从草稿、候选正文或场景编排包中生成 `characters/state_patches/{scene_id}_state_patch.md` / `.json`。该产物只记录候选变化，包括已知事实、资源、位置、健康、弧光和关系变化。

人物档案不得被模型或草稿直接改写。重大转折必须先进入 state patch，经过人工确认后，再由后续受控写回命令处理。
